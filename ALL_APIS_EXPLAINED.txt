================================================================================
PORTFOLIO BUILDER — COMPLETE API LIST & SYSTEM FLOW
Server 1 (Port 3001) | Server 2 (Port 3000)
================================================================================

1. AUTHENTICATION (Server 1)
----------------------------------------
POST /api/auth/register
- Role: Register new user.
- Flow: Salts/hashes password -> Saves to DB -> Returns JWT + sets HttpOnly cookie.

POST /api/auth/login
- Role: Authenticate user.
- Flow: Checks bcrypt hash -> Checks Redis for failed login attempts (rate-limit) -> Returns JWT.

POST /api/auth/logout
- Role: Revoke access.
- Flow: Blacklists the JWT unique ID (jti) in Redis until expiry.

POST /api/auth/refresh
- Role: Renew expired access token.
- Flow: Checks Refresh Token cookie -> Verifies against secret -> Issues new Access Token.


2. PROJECT MANAGEMENT (Server 1)
----------------------------------------
GET /api/projects/templates
- Role: Get portfolio gallery.
- Flow: Checks Redis cache (10 min) -> Fallback to DB -> Returns list (filters gitRepoUrl).

GET /api/projects
- Role: Get user's dashboard.
- Flow: Fetches user's projects from DB -> Sorts by most recently opened.

POST /api/projects/create
- Role: Start a new portfolio.
- Flow: DB Insert (READY) -> Git clone (no install) -> Delete .git (safety). Very fast.

GET /api/projects/:id/files
- Role: Populate IDE sidebar.
- Flow: Recursive walk of instance directory -> Returns JSON tree (skips node_modules).

GET /api/projects/:id/files/*
- Role: Open file in Monaco editor.
- Flow: Resolve path -> Assert safe (inside root) -> Read content -> Return content.

PUT /api/projects/:id/files/*
- Role: Save work.
- Flow: Write to disk -> Update DB timestamp -> Touch Redis activity key.


3. SNAPSHOTS & RECOVERY (Server 1)
----------------------------------------
GET /api/projects/:id/snapshots
- Role: View history.
- Flow: Returns list of labeled manual/auto saves (excludes content for speed).

POST /api/projects/:id/snapshots
- Role: Manual "Checkpoint".
- Flow: Walks /src -> Stringifies all code files into JSON blob -> Save to DB Snapshot row.

POST /api/projects/:id/snapshots/:sid/restore
- Role: Rollback project.
- Flow: Create safety PRERESTORE save -> Wipe current files -> Loop JSON blob and recreate files.


4. PREVIEW SYSTEM (Server 1 <-> Server 2)
------------------------------------------
POST /api/preview/:id/start
- Role: "Run" the website.
- S1 Flow: Checks Redis for port -> If absent, calls Server 2 /assign.
- S2 Flow: Find free port (4001+) -> Check node_modules (npm install if missing) -> Spawn Vite.

DELETE /api/preview/:id/stop
- Role: Free resources.
- S1 Flow: Calls Server 2 /release -> Clears Redis cache.
- S2 Flow: Kills PID -> Marks port as 'free' in the memory pool.


5. RESUME & AI (Server 1)
----------------------------------------
POST /api/resume/parse
- Role: Convert PDF to Data.
- Flow: Multer upload -> pdfjs-dist extraction -> Ollama/Claude refinement -> Returns JSON.

POST /api/resume/apply/:pid
- Role: Auto-generate portfolio content.
- Flow: Load project files -> Replace {{NAME}}, {{SKILLS}} placeholders with real data.


6. DEPLOYMENT (Server 1)
----------------------------------------
POST /api/deploy/:id
- Role: Go live (Internet).
- Flow: Push task to Bull Queue -> Worker runs 'npm run build' -> Uploads /dist to CDN.
